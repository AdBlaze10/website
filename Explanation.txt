# 🧠 Full Technical Explanation of the Symptom Checker Project

## 1. Overview

This project is a **web-based symptom checker built using Django**. It allows users to select symptoms through a web interface, processes those symptoms using a rule-based system, and returns possible conditions along with a triage level.

The system is composed of:

* **Frontend (HTML templates)** → collects user input and displays results
* **Backend (Django views)** → processes requests and controls logic flow
* **Data layer (`database.py`)** → stores symptoms, conditions, and rules
* **Logic layer (functions in `views.py`)** → computes results

# 🧠 3. Data Layer (`database.py`)

This file acts as a **custom knowledge base**.

## Example structure:

```python
CONDITIONS = [
    {
        "name": "Flu",
        "symptoms": {"fever": 3, "cough": 2},
        "triage": "medium",
        "advice": "Rest and hydrate"
    }
]
```

### What each part means:

* `"name"` → condition label shown to user
* `"symptoms"` → dictionary mapping symptoms to weights
* `"triage"` → urgency level
* `"advice"` → recommended action

👉 This is essentially a **rule-based dataset**.
